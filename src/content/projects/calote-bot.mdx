## O Problema

Um amigo que trabalha com vendas tinha um problema: clientes que não pagavam. Não por má-fé, mas por esquecerem. Ele passava horas por dia mandando mensagens de cobrança manualmente, uma por uma, copiando e colando.

Eu disse: "cara, isso é trabalho de bot".

## A Solução

**Calote-bot** é um bot que automatiza cobranças via WhatsApp com:

- Templates de mensagem personalizáveis
- Variáveis dinâmicas (nome, valor, data)
- Agendamento de envios
- CLI interativo para gerenciamento

O nome? Ironia. O bot é pra evitar calotes, não pra dar.

## Arquitetura

O bot é simples por design. Nada de cloud, database, ou microserviços — roda na máquina do usuário como um processo Node.js.

## Templates Dinâmicos

O sistema de templates permite criar mensagens com variáveis:

```typescript
// templates/template.types.ts
interface Template {
  id: string;
  name: string;
  message: string;
  variables: string[];
}

interface Contact {
  name: string;
  phone: string;
  debt: number;
  dueDate: string;
  [key: string]: string | number; // Variáveis customizadas
}
```

Exemplo de template:

```json
{
  "id": "cobranca-inicial",
  "name": "Cobrança Inicial",
  "message": "Olá {{name}}! 👋\n\nNotamos que o pagamento de R$ {{debt}} com vencimento em {{dueDate}} ainda não foi registrado.\n\nPrecisa de ajuda para regularizar? Estamos à disposição!\n\n📱 Responda esta mensagem ou ligue para nosso suporte.",
  "variables": ["name", "debt", "dueDate"]
}
```

### Engine de Templates

A substituição de variáveis é feita com regex simples:

```typescript
// services/template.service.ts
export function renderTemplate(template: Template, contact: Contact): string {
  let message = template.message;

  for (const variable of template.variables) {
    const value = contact[variable];
    const placeholder = new RegExp(`{{${variable}}}`, "g");

    // Formatar valores especiais
    let formattedValue: string;
    if (variable === "debt" && typeof value === "number") {
      formattedValue = value.toLocaleString("pt-BR", {
        style: "currency",
        currency: "BRL",
      });
    } else if (variable === "dueDate") {
      formattedValue = dayjs(value as string).format("DD/MM/YYYY");
    } else {
      formattedValue = String(value);
    }

    message = message.replace(placeholder, formattedValue);
  }

  return message;
}
```

## Agendamento

O bot usa node-schedule para agendar envios:

```typescript
// services/scheduler.service.ts
import schedule from "node-schedule";
import { sendMessage } from "./whatsapp.service";

interface ScheduledJob {
  id: string;
  contact: Contact;
  template: Template;
  scheduledFor: Date;
  job: schedule.Job;
}

const scheduledJobs: Map<string, ScheduledJob> = new Map();

export function scheduleMessage(
  contact: Contact,
  template: Template,
  dateTime: Date,
): string {
  const jobId = `${contact.phone}-${Date.now()}`;

  const job = schedule.scheduleJob(dateTime, async () => {
    try {
      const message = renderTemplate(template, contact);
      await sendMessage(contact.phone, message);

      console.log(chalk.green(`✓ Mensagem enviada para ${contact.name}`));

      // Remover job após execução
      scheduledJobs.delete(jobId);
    } catch (error) {
      console.log(chalk.red(`✗ Falha ao enviar para ${contact.name}`));
    }
  });

  scheduledJobs.set(jobId, {
    id: jobId,
    contact,
    template,
    scheduledFor: dateTime,
    job,
  });

  return jobId;
}

// Agendar múltiplos contatos com intervalo
export async function scheduleMultiple(
  contacts: Contact[],
  template: Template,
  startTime: Date,
  intervalMinutes: number,
): Promise<void> {
  let currentTime = dayjs(startTime);

  for (const contact of contacts) {
    scheduleMessage(contact, template, currentTime.toDate());
    currentTime = currentTime.add(intervalMinutes, "minute");
  }

  console.log(chalk.green(`\n✓ ${contacts.length} mensagens agendadas`));
  console.log(
    chalk.gray(`  Início: ${dayjs(startTime).format("DD/MM HH:mm")}`),
  );
  console.log(chalk.gray(`  Fim: ${currentTime.format("DD/MM HH:mm")}`));
}
```

## Integração WhatsApp

O venom-bot faz a automação do WhatsApp Web:

```typescript
// services/whatsapp.service.ts
import venom from "venom-bot";

let client: venom.Whatsapp;

export async function initWhatsApp(): Promise<void> {
  client = await venom.create({
    session: "calote-bot",
    multidevice: true,
    headless: true,
    useChrome: true,
    logQR: true, // Mostra QR no terminal
  });

  console.log(chalk.green("✓ WhatsApp conectado"));
}

export async function sendMessage(
  phone: string,
  message: string,
): Promise<void> {
  // Normalizar número brasileiro
  const normalizedPhone = normalizePhone(phone);

  // Verificar se número existe no WhatsApp
  const isValid = await client.checkNumberStatus(normalizedPhone);

  if (!isValid.numberExists) {
    throw new Error(`Número não encontrado: ${phone}`);
  }

  await client.sendText(normalizedPhone, message);
}

function normalizePhone(phone: string): string {
  // Remove tudo que não é número
  const digits = phone.replace(/\D/g, "");

  // Adiciona código do país se necessário
  if (digits.length === 11) {
    return `55${digits}@c.us`;
  } else if (digits.length === 13 && digits.startsWith("55")) {
    return `${digits}@c.us`;
  }

  throw new Error(`Formato de telefone inválido: ${phone}`);
}
```

## Decisões Técnicas

### Por que venom-bot?

- Não precisa de API oficial do WhatsApp (que é cara e burocrática)
- Simula um navegador real
- Mantido ativamente
- Suporta multi-device

### Por que não usar uma fila (Redis/RabbitMQ)?

- Complexidade desnecessária para o caso de uso
- Roda local, não precisa de infra
- node-schedule resolve o agendamento

### Por que JSON para dados?

- Simplicidade
- Usuário pode editar manualmente se quiser
- Sem necessidade de migrations

## Limitações Conhecidas

1. **Rate limiting**: WhatsApp pode bloquear se enviar muitas mensagens rápido
2. **Sessão expira**: Precisa escanear QR de novo periodicamente
3. **Não oficial**: Pode quebrar se WhatsApp mudar a API

## O que aprendi

1. **Automação de WhatsApp é frágil** — APIs não oficiais dependem de engenharia reversa
2. **CLI bem feito é UX** — inquirer transforma scripts em produtos
3. **Documentação com personalidade engaja** — o README do Calote-bot é o mais compartilhado que eu já fiz
4. **Resolver problemas pequenos vale a pena** — meu amigo economiza 2h por dia

## The Problem

During my cybersecurity studies, I noticed a significant gap: most educational materials focused on theory or showed isolated attacks, without demonstrating how defenses work together in a real system.

The question I wanted to answer was: **how do you build a backend that's genuinely hard to break into?**

## Overview

The NestJS Attack & Defense Lab is an educational platform that simulates attack and defense scenarios in real time. It implements a simplified SOC (Security Operations Center) where:

- **Red Team** runs automated attacks via Python scripts
- **Blue Team** monitors and responds through a React dashboard
- **Backend** implements 7 layers of defense in depth

## Defense Architecture

The architecture follows the **defense in depth** principle: each layer adds a barrier an attacker needs to overcome:

### Layer 2: WAF (Web Application Firewall)

Pattern matching to detect known malicious payloads:

```typescript
@Injectable()
export class WafGuard implements CanActivate {
  private readonly sqlPatterns = [
    /(\b(SELECT|INSERT|UPDATE|DELETE|DROP|UNION)\b)/gi,
    /(\b(OR|AND)\b\s+\d+\s*=\s*\d+)/gi,
    /(--|\#|\/\*)/g,
  ];

  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    const payload = JSON.stringify(request.body);
    if (this.detectSqlInjection(payload)) {
      this.logThreat(request, "SQL_INJECTION");
      throw new ForbiddenException("Suspicious request blocked");
    }
    return true;
  }
}
```

## What I Learned

Building a system to attack yourself is a completely different way of thinking about security. You stop thinking "what could go wrong" and start thinking "how would I exploit this".

The most valuable insight: **most vulnerabilities come from trusting user input too much**. The WAF and input validation alone blocked ~70% of the test payloads.
